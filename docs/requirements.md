# PortaFile 機能要件

## 動作環境

- Windows 11 64bit
- C# WPF
- .NET 10

## 目的

COMポートを利用して、2台のWindows PC間でファイルまたはフォルダを送受信する。
双方でPortaFileを起動しておき、送信側ウィンドウへファイルまたはフォルダをドロップすると転送を開始する。

## 通信形態

- 通信相手は常に1対1。
- 全二重と半二重を切り替え可能。
- 半二重では、送受信制御方式を以下から選択可能。
  - ドライバ任せ
  - RTS制御
- RTS制御では送信直前にRTSをON、送信完了待ち後にRTSをOFFにする。

## シリアル設定

- COMポート
- ボーレート
- パリティ
- 全二重 / 半二重
- 半二重方向制御

ボーレート候補:

- 9600
- 19200
- 38400
- 57600
- 115200
- 230400
- 460800

- 1000000
- 2000000
- 3000000

## 送信開始と競合制御

- ウィンドウへのドラッグ＆ドロップで送信開始。
- ファイル、複数ファイル、フォルダを対象にする。
- フォルダは相対パス構造を保持して送信する。
- 両端で同時にドロップした場合は早い者勝ち。
- 衝突時の安定化のため、起動時にランダム生成したNodeIdでタイブレークする。

## ファイルメタデータ

送信前にマニフェストを送信する。

- ルート名
- ファイル数
- 総バイト数
- 各ファイルの相対パス
- 各ファイルのサイズ
- 各ファイルの更新日時UTC
- 各ファイル全体のCRC-32

## 受信保存先

- 実行ファイル直下に `Downloads` フォルダを自動作成する。
- 受信ファイルは `Downloads` 配下に保存する。
- フォルダ送信時は `Downloads\<送信元フォルダ名>\...` に展開する。
- 受信中は `.part` ファイルに保存する。
- ファイル全体CRC-32検証後に正式名へリネームする。
- 同名ファイルが存在する場合は上書きせず、`file (1).txt`、`file (2).txt` のようにリネームする。

## パケット転送

- データブロックサイズは10KiB、つまり10,240 bytes。
- パケット単位でCRC-32を付加する。
- ファイル全体にもCRC-32を付加する。
- DATAパケットは連番で管理する。

## ARQ / 再送

ACK/NAK方式で再送制御する。

再送条件:

- パケットCRC不一致
- タイムアウト
- 想定外シーケンス番号

最大再送回数:

- 初期値は10回。
- 最大再送回数を超えた場合はユーザーに続行確認する。
- 続行する場合は再送カウントをリセットして継続する。
- 中止する場合は転送失敗として終了する。

タイムアウト:

- 10KiB送信時間 + 2秒を基本に自動計算する。
- 最低値は3秒。

## キャンセル

- 送信側・受信側のどちらからでも転送中キャンセル可能。
- キャンセル時はCANCELパケットを相手に送信する。
- MVPでは受信中の `.part` ファイルを削除する。

## 進捗表示

Windowsのデフラグ画面風に、転送ブロックの状態を色分け表示する。

ブロック状態:

- 未送信
- 送信中
- 送信成功
- 再送中
- 失敗
- 検証中

表示項目:

- 現在のファイル
- 全体進捗
- 現在ファイル進捗
- 転送速度
- 残り時間
- 再送回数
- エラー回数
