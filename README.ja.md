# PortaFile

PortaFile は、COM ポートを利用して 2 台の Windows PC 間でファイルやフォルダを転送する WPF アプリケーションです。

双方の PC で PortaFile を起動し、送信側のウィンドウへファイルまたはフォルダをドラッグ＆ドロップすると転送を開始します。フォルダ転送時は相対パス構造を保持し、CRC-32 によってデータの整合性を検証します。

## 主な機能

- ファイル、複数ファイル、フォルダのドラッグ＆ドロップ送信
- フォルダ転送時の相対パス構造保持
- COM ポートを利用した 1 対 1 のシリアル通信
- 全二重通信と半二重通信の切り替え
- 半二重通信時の RTS 方向制御
- パケット単位およびファイル単位の CRC-32 検証
- ACK / NAK による再送制御
- 送信側・受信側どちらからでも転送キャンセル可能
- 受信先に同名ファイルがある場合は上書きせず自動リネーム
- Windows のデフラグ画面風のブロック進捗表示

## 動作環境

- Windows 11 64 bit
- .NET 10 SDK
- 2 台の PC 間を接続するシリアル通信環境

## ビルド

```powershell
dotnet build
```

このプロジェクトは `net10.0-windows` を対象とした WPF アプリケーションです。

## 使い方

1. 2 台の PC で PortaFile を起動します。
2. それぞれの PC で COM ポートとシリアル設定を選択します。
3. 通信モードを選択します。
   - 全二重
   - 半二重
   - 半二重 + RTS 制御
4. 送信側のウィンドウへ、ファイルまたはフォルダをドラッグ＆ドロップします。
5. 受信したファイルは、実行ファイルと同じ場所に作成される `Downloads` フォルダへ保存されます。

受信中のファイルは一時的に `.part` ファイルとして保存されます。CRC-32 検証に成功すると、正式なファイル名へリネームされます。

## シリアル設定

PortaFile では以下の設定を扱います。

- COM ポート
- ボーレート
- パリティ
- 全二重 / 半二重
- 半二重時の方向制御

ボーレート候補:

- 115200
- 230400
- 460800
- 921600
- 1000000
- 2000000
- 3000000

## プロジェクト構成

```text
PortaFile/
├─ Models/
├─ Protocol/
├─ Services/
├─ Transfer/
├─ ViewModels/
├─ Views/
└─ docs/
```

- `Views/`: WPF の画面定義。`MainWindow.xaml.cs` は ViewModel 生成とドラッグ＆ドロップなど View 固有イベントの橋渡し
- `ViewModels/`: 画面状態、コマンド、バインディング用モデル
- `Models/`: ViewModel から参照する表示・選択用モデル
- `Protocol/`: パケット種別、パケットのエンコード / デコード、CRC-32
- `Services/`: シリアルポート設定、通信処理、Windowsダイアログ連携
- `Transfer/`: マニフェスト作成、保存先パス解決、進捗管理、転送エンジン
- `docs/requirements.md`: 機能要件

## プロトコル概要

PortaFile は以下のような種別のパケットを送受信します。

- `HELLO`
- `SEND_REQUEST`
- `READY`
- `BUSY`
- `MANIFEST`
- `FILE_START`
- `DATA`
- `ACK`
- `NAK`
- `FILE_END`
- `FILE_OK`
- `FILE_ERROR`
- `TRANSFER_END`
- `CANCEL`
- `ERROR`

データパケットは 10 KiB 単位で送信されます。CRC 不一致、タイムアウト、想定外のシーケンス番号が発生した場合は再送を行います。
