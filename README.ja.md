# PortaFile

[English](./README.md) | **日本語**

PortaFile は、COM ポートを利用して 2 台の Windows PC 間でファイルやフォルダを転送する WPF アプリケーションです。
双方の PC で PortaFile を起動し、送信側のウィンドウへファイルやフォルダをドラッグ＆ドロップするだけで転送を開始します。フォルダ転送時は相対パス構造を保持し、CRC-32 によってデータの整合性を検証します。

## 主な機能

- **ドラッグ＆ドロップによる簡単送信**: ファイル、複数ファイル、フォルダをウィンドウへドラッグ＆ドロップするだけで転送できます。
- **相対パス構造の維持**: フォルダを転送する際、階層構造を保持したまま転送されます。
- **1対1シリアル通信**: COM ポートを利用したシリアル通信により、インターネット不要で安全にデータをやり取りします。
- **転送信頼性モードの選択**: ACK/NAKによる高信頼な再送制御（ARQ）を行うモードと、応答確認なしで一方向に高速転送するモード（One-Way）を選択可能です。
- **柔軟な通信設定**: 全二重通信と半二重通信の切り替え、半二重通信時の RTS 方向制御が可能です。
- **強固な整合性検証**: パケット単位およびファイル単位での CRC-32 検証を実施します。
- **双方向からのキャンセル**: 送信側・受信側のどちらからでも、いつでも転送を安全にキャンセルできます。
- **同名ファイル対策**: 受信先に同名ファイルが存在する場合、上書きせず自動的にリネームして保存します。
- **デフラグ風の進捗表示**: Windows のクラシックなデフラグ画面を彷彿とさせるブロックデザインで転送進捗を表示します。

## 動作環境

- **OS**: Windows 10 / 11 64-bit
- **フレームワーク/SDK**: .NET 10 SDK
- **ハードウェア**: 2台の PC 間を接続するシリアル通信ケーブル（クロスケーブルなど）

## 使い方

1. 転送を行う両方の PC で PortaFile を起動します。
2. それぞれの PC で使用する **COM ポート** と **シリアル通信設定** を選択します。
3. 転送の **信頼性モード** (ARQ / One-Way) を選択します。
4. **通信モード** を以下から選択します：
   - 全二重
   - 半二重
   - 半二重 + RTS 制御
5. 送信側のウィンドウへ、ファイルまたはフォルダをドラッグ＆ドロップします。
6. 受信したファイルは、実行ファイルと同じ場所に作成される `Downloads` フォルダへ自動保存されます。

*※受信中のファイルは一時的に `.part` の拡張子で保存され、CRC-32 検証に成功したタイミングで正式なファイル名へリネームされます。*

## シリアル構成

PortaFile では以下の通信設定をサポートしています。

- **COMポート選択**
- **パリティビット設定**
- **信頼性モード**: ARQ（応答確認・再送制御あり） / One-Way（一方向垂れ流し）
- **デュプレックスモード**: 全二重 / 半二重 / 半二重RTS方向制御
- **サポートするボーレート候補**:
  - 115200 / 230400 / 460800 / 921600 / 1000000 / 2000000 / 3000000

## プロトコル概要

PortaFile は独自に設計されたプロトコルを用いて、パケット単位でデータの送受信と状態管理を行います。

### 送受信パケット種別
`HELLO` / `SEND_REQUEST` / `READY` / `BUSY` / `MANIFEST` / `FILE_START` / `DATA` / `ACK` / `NAK` / `FILE_END` / `FILE_OK` / `FILE_ERROR` / `TRANSFER_END` / `CANCEL` / `ERROR` / `DATA_BATCH_CHECK` / `DATA_BATCH_ACK`

### 転送プロセス
データパケットは **10 KiB** 単位で分割して送信されます。パケットごとに CRC-32 値を付与し、受信側でのチェックで CRC 不一致・タイムアウト・想定外のシーケンス番号が発生した場合は、ACK/NAK 制御（ARQモード時）によって自動的に再送処理が走ります。

## プロジェクト構成

```text
PortaFile/
├── Models/        # 表示・選択用データモデル
├── Protocol/      # パケット定義、エンコード/デコード、CRC-32計算
├── Services/      # シリアルポート設定、シリアル通信処理、Windows API連携
├── Transfer/      # マニフェスト作成、ファイルパス解決、転送エンジン、進捗管理
├── ViewModels/    # MVVMパターンのViewModel、コマンドおよびバインディング
├── Views/         # WPFによるUI表示定義（MainWindow等）
└── docs/          # 機能要件ドキュメント (requirements.md)
```
